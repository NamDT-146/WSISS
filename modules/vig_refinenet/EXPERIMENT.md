# Experimental Setup

## 1. Task Definition

We evaluate the effectiveness of the proposed **multi-stage GNN refinement module** for improving **weakly supervised segmentation signals**.

Given:

* an RGB image
* a weak supervision signal (mask / box / point)

the goal is to predict a refined binary segmentation mask.

---

## 2. Weak Supervision Types

We consider **four types of weak signals**, synthetically generated from ground-truth annotations to ensure fair and controlled evaluation.

### 2.1 Weak Signal Categories

| Type          | Description                | Purpose                  |
| ------------- | -------------------------- | ------------------------ |
| **Good Mask** | Slightly corrupted GT mask | Upper-bound refinement   |
| **Poor Mask** | Heavily corrupted GT mask  | Robustness               |
| **Box**       | Bounding box supervision   | Detection → segmentation |
| **Point**     | 1–3 foreground points      | Extreme weak supervision |

This design follows standard practice in weakly supervised and interactive segmentation literature.

---

### 2.2 Weak Signal Generation

All weak signals are generated **only for training and evaluation**, not for supervision beyond initialization.

#### (1) Good Mask

* Generated by mild erosion/dilation of ground-truth masks
* Boundary noise is introduced to simulate imperfect predictions

Target initial IoU: **~80–90%**

#### (2) Poor Mask

* Generated by aggressive morphological operations and random region dropout
* Contains significant boundary and interior errors

Target initial IoU: **~40–60%**

#### (3) Box

* Axis-aligned bounding boxes extracted from ground-truth masks
* Variants include:

  * tight box
  * loose box (+10–20% expansion)

Initial mask is constructed as:

* inside box: foreground / unknown
* outside box: background

#### (4) Point

* 1–3 random foreground points sampled from GT mask
* All other pixels treated as unknown

---

## 3. Dataset Split Protocol

The dataset is split **once and fixed** for all experiments.

| Split    | Ratio | Usage                           |
| -------- | ----- | ------------------------------- |
| Training | 75%   | Model training                  |
| Testing  | 25%   | Final evaluation and comparison |

* No overlap between splits
* The same test set is used for all methods and all weak-signal types

---

## 4. Compared Methods

For each weak supervision type, we compare:

### 4.1 Methods

| Method       | Description                                    |
| ------------ | ---------------------------------------------- |
| **Original** | Initial weak signal (no refinement)            |
| **DenseCRF** | Dense Conditional Random Field post-processing |
| **Ours**     | Proposed multi-stage GNN refinement            |

DenseCRF serves as a **classical non-learning refinement baseline**, widely used in segmentation literature.

---

## 5. Training Protocol

### 5.1 Model Components

* **Backbone**: ResNet-50 pretrained on ImageNet
* **Feature extraction**:

  * multi-scale features (7×7, 14×14, 28×28)
* **Refinement module**:

  * 3-stage GNN
  * parent → child feature inheritance
  * progressively finer graph resolution
* **Prediction head**:

  * 1×1 convolution producing final binary mask

---

### 5.2 Training Details

* Training uses **only the 75% training split**
* Weak signals are provided as **input only**
* Supervision is applied **only on the final output mask**

#### Loss Function:

```text
Binary Cross-Entropy + Dice Loss
```

#### Optimization:

* Optimizer: Adam
* Learning rate: fixed / cosine decay (to be specified)
* Batch size: (dataset dependent)

---

## 6. Evaluation Protocol

Evaluation is performed **only on the held-out 25% test set**.

### 6.1 Metrics

We report:

* **IoU (Intersection-over-Union)**
* **Dice coefficient**

Metrics are computed:

* per image
* then averaged over the test set

---

### 6.2 Evaluation Comparisons

For each weak-signal type, we compare:

| Input Type | Original | DenseCRF | Ours |
| ---------- | -------- | -------- | ---- |
| Good Mask  | ✓        | ✓        | ✓    |
| Poor Mask  | ✓        | ✓        | ✓    |
| Box        | ✓        | ✓        | ✓    |
| Point      | ✓        | ✓        | ✓    |

For **Good Mask and Poor Mask**, we explicitly report:

* original weak mask performance
* improvement after CRF
* improvement after our refinement

---

## 7. Experimental Scripts and Code Structure

To ensure clarity and reproducibility, the implementation is organized into **modular components**.

### 7.1 Core Code Modules

#### (1) Model Code

* Backbone (ResNet-50)
* Multi-stage GNN refinement
* Feature inheritance modules
* Mask prediction head

#### (2) Weak Signal Generation

* Good mask generator
* Poor mask generator
* Box generator
* Point generator

Each generator is deterministic given a random seed.

#### (3) DenseCRF Module

* Standard DenseCRF implementation
* Used only during evaluation
* No learning involved

---

### 7.2 Main Scripts

#### Script 1 — Training

```text
train.py
```

Responsibilities:

* load training split
* generate weak signals on-the-fly
* train the refinement model
* save checkpoints

---

#### Script 2 — Evaluation

```text
evaluate.py
```

Responsibilities:

* load trained model
* load test split
* generate weak signals
* run:

  * original mask
  * DenseCRF
  * proposed method
* compute IoU and Dice
* generate comparison tables

---

## 8. Reporting and Analysis

Final results are reported as:

* quantitative tables (IoU / Dice)
* per weak-signal type
* qualitative visualizations (optional but recommended)

This setup allows a **clean, controlled, and fair comparison** between classical CRF-based refinement and the proposed learning-based approach.

---

## 9. Summary

This experimental design:

* isolates refinement capability from segmentation quality
* supports fair comparison with DenseCRF
* evaluates robustness across multiple weak-supervision regimes
* ensures full reproducibility
